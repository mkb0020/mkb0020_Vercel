Privacy Policy

01 · Who We Are

CatForce is a personal desktop application developed by mkb0020, an independent game and software developer. This policy describes how CatForce handles data when you use the application to manage social media content.

CatForce is a private tool — it is not a public service, and access is limited to its developer and authorized users.

02 · Data We Collect

⊘ CatForce does not sell, share, or monetize your data. Ever.

The following data is collected solely to operate CatForce's content scheduling features:

03 · How We Use Your Data

Data collected by CatForce is used exclusively for:

  • Displaying available posts in the CatForce desktop application.
  • Uploading video drafts to your connected TikTok inbox.
  • Tracking posting history to avoid repeating recent content.
  • Refreshing OAuth tokens to maintain an active TikTok connection.

No data is used for advertising, analytics sold to third parties, or any purpose beyond operating CatForce itself.

04 · TikTok Integration

When you connect a TikTok account to CatForce, the following occurs:

  • You are redirected to TikTok's official authorization page in your system browser.
  • CatForce requests only the scopes user.info.basic and video.upload.
  • TikTok issues an access token and refresh token, which are stored securely in a private Supabase database.
  • These tokens are never stored in the CatForce desktop application itself, and never transmitted to any third party other than TikTok's official API.

Videos are sent to your TikTok inbox as private drafts. CatForce does not publish content publicly — you review and publish each draft manually in the TikTok app.

You can revoke CatForce's TikTok access at any time via TikTok Settings → Manage App Permissions.

05 · Data Storage & Security

All CatForce data is stored in a private Supabase project with the following protections:

  • OAuth tokens are stored server-side only — they are never exposed to the desktop application's frontend.
  • All API access to Supabase from the CatForce backend uses a service role key, restricted to the server environment.
  • Row Level Security (RLS) is enabled on token tables — no anonymous or public access is permitted.
  • The backend runs on Vercel with HTTPS enforced on all endpoints.

06 · Data Retention

Data is retained for as long as CatForce is in active use. You may request deletion of your data at any time by contacting the developer. OAuth tokens can be invalidated immediately by revoking app access through TikTok's settings.
